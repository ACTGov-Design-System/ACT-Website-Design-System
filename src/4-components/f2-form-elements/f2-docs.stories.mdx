								<h3>Using forms</h3>
								<p>Forms comprise of the form itself and multiple form inputs. Each form is broken down into one or more fieldsets that groups the form elements into logical chunks that make sense to the user, for example, a delivery address.</p>
								<p>Form elements are split into two key input types:</p>
								<ul>
									<li>Freeform inputs - Free form text inputs are the most used components in forms. Use text fields and text areas for freeform entry. Text fields are used to input a short (single line) amount of freeform text. Text areas are used to input multiple lines of text, for example, a comments field. Use text areas when you expect longer entries.</li>
									<li>Selection inputs - Selection inputs are used to select an option from a predefined list of available options. Selection options may be single-select or multi-select. Use a radio button list or dropdowns for single selections and checkboxes for multi-selections.</li>
								</ul>
								<h4>Do:</h4>
								<ul>
									<li>keep it one column - multiple columns easily disrupt a user’s vertical momentum</li>
									<li>use field length input constraints - employ this for fields that have a defined character count like phone numbers and postcodes</li>
									<li>mark required vs. optional form fields - you should always mark your form fields as “required” or “optional”. You can do this by writing “required” or “optional” in a small font next to your field, or by placing an asterisk next to your required fields</li>
									<li>use top align labels - users complete top aligned labeled forms at a much higher rate than left-aligned labels. Top aligned labels also translate well on mobile.</li>
									<li>make CTAs descriptive, state the intent of the action</li>
									<li>specify errors inline, show the user where the error occurred and provide a reason.</li>
								</ul>
								<h4>Do not</h4>
								<ul>
									<li>use all caps</li>
